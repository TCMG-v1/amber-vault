# Velmic Symbol-Language Merch Bible

## Working premise

Velmic merch should not present the glyphs as decorative marks. The brand advantage is that the symbols behave like a language: each mark carries a function, each color encodes a state, and each product teaches the buyer how to compose meaning.

The collection should feel like a field kit for operators, players, archivists, and builders. A sticker is not only a sticker. It is a portable symbol, a state marker, a game token, a seal, a patch of grammar, or a small machine instruction.

## Core brand idea

**Velmic is a symbolic operating language for state, signal, and passage.**

This should drive every design decision. If an element does not clarify the grammar, the state system, the glyph role, or the ritual of use, remove it.

### Positioning language

- **Primary line**: Symbols that do work.
- **Secondary line**: Wear the grammar.
- **System line**: Not icons. Operators.
- **Collector line**: A language for sealed things, moving things, and returning things.
- **Game line**: Glyphs for state, signal, and passage.

### Brand promise

Velmic gives players, builders, and collectors a visible grammar for marking intent, tracking state, and composing meaning across physical and digital objects.

## Voice and tone

### Voice attributes

#### Systemic

- **We are**: precise, rule-aware, and compositional.
- **We are not**: vague, mystical, or purely ornamental.
- **This sounds like**: “Each glyph carries a role. Each color activates a state.”
- **This does not sound like**: “Cool ancient-looking symbols for your laptop.”

#### Archival

- **We are**: artifact-conscious, catalogued, and canonical.
- **We are not**: dusty, academic for its own sake, or inaccessible.
- **This sounds like**: “Filed under G07, Pause Register, transition arc.”
- **This does not sound like**: “Random drop from the vault.”

#### Playable

- **We are**: game-facing, usable, and mechanically legible.
- **We are not**: passive lore with no function.
- **This sounds like**: “Use Amber to suspend the effect. Use Cyan to return it to play.”
- **This does not sound like**: “The glyph has vibes.”

#### Sealed

- **We are**: deliberate, quiet, and encoded.
- **We are not**: loud, gimmicky, or overexplained.
- **This sounds like**: “Mark the object. Bind the state. Move when ready.”
- **This does not sound like**: “Look at this wild graphic.”

## Visual identity principles

### Design posture

The visual system should be restrained and infrastructural. Let glyph geometry, spacing, slot IDs, and state color do the work. Avoid fantasy-rune styling, fake occult texture, generic cyberpunk clutter, distressed type, and decorative symbols that do not belong to the actual grammar.

### Layout rules

- **Glyph first**: The symbol is always the dominant object.
- **Metadata second**: Slot ID, state, semantic role, and arc position support the symbol.
- **Decoration last**: Only add marks that behave like registration, indexing, printing, sealing, or game-state notation.
- **Negative space**: Treat empty space as the field that lets the glyph carry signal.

### Recommended label syntax

```text
VELMIC / G07
STATE: 2 / PAUSE
ROLE: INTERRUPTION FIELD
ARC: TRANSITION
```

For foundational forms:

```text
VELMIC / F02
FORM: RAY
FUNCTION: DIRECTED EXTENSION
STATE: 1 / RETURN
```

The 0-2-1 color system should be treated as semantic encoding, not colorway decoration.

## Glyph families

### Primitives

The primitives are the best first merch line because they are simple, legible, inexpensive to produce, and easy to teach.

| Slot | Form | Function | Merch interpretation | Game interpretation |
|---|---|---|---|---|
| F1 | Line | Unbounded extension, carrier signal, raw potential | Continuity mark, base signal, minimal seal | Extend range, duration, or continuity |
| F2 | Ray | Directed extension from fixed origin | Intent mark, forward motion, commitment | Commit direction, target, or path |
| F3 | Bilateral | Bidirectional relation, dialogue, exchange | Pairing mark, bridge, trade signal | Exchange state between two entities |
| F4 | Pending extraction | Closed or terminated form to confirm | Completion mark once verified | Lock, terminate, seal, or resolve once verified |

### Derived alphabet

The 22-glyph Mnemosyne alphabet should be positioned as the collector and composition layer. Buyers should understand that each glyph has a slot, a function, and a position in the larger arc.

Recommended framing:

- **Early glyphs**: origin, emergence, seed, potential.
- **Middle glyphs**: transition, relation, complication, crossing.
- **Late glyphs**: arrival, return, resolution, Anya.

Until the glyphs are extracted, do not overname the 22 positions. Use slot IDs and reserve final naming for the production glyph sheet.

## Game-focused system

The game-facing merch should make Velmic feel playable immediately. Treat glyphs as rule components.

### Core game verbs

| Verb | Symbolic basis | Use in play |
|---|---|---|
| Carry | Blue / 0 | Store an effect, reserve energy, hold a state for later |
| Pause | Amber / 2 | Suspend, interrupt, protect, delay, or freeze a sequence |
| Return | Cyan / 1 | Reactivate, recurse, resolve, or bring a state back into play |
| Extend | Line | Increase range, duration, continuity, or connection |
| Commit | Ray | Choose direction, lock intent, set target, move forward |
| Exchange | Bilateral | Swap, mirror, negotiate, trade, or link two entities |
| Seal | F4 pending | Close, terminate, bind, or make permanent once confirmed |

### Productized game objects

- **State token stickers**: Blue, Amber, and Cyan markers for cards, boards, notebooks, laptops, and prototype components.
- **Glyph cards**: Each card contains one glyph, one semantic function, one game verb, one state interaction, and one short example.
- **Rule inserts**: Small cipher cards included with every order so buyers learn how to compose meaning.
- **Faction patches**: Howlbox, Stellaria, Neutral, and Native Velmic treated as schools, registers, or print editions.
- **Prototype overlays**: Clear vinyl glyphs for modifying physical game components without permanently redesigning them.

### Sample card format

```text
G07 / PAUSE REGISTER

Function:
Suspends a moving effect without destroying its origin.

Game use:
Place Amber G07 on an active card. The effect remains present but cannot resolve until returned.

State path:
Blue holds → Amber pauses → Cyan returns.
```

## Product architecture

### Launch tier

#### Primitive Pack

- **Contents**: F1-F4 in Blue, Amber, and Cyan.
- **Role**: Entry product and teaching object.
- **Audience**: Designers, players, collectors, notebook/laptop users.
- **Format**: Die-cut stickers or clear vinyl decals.
- **Copy**: “Four base forms. Three states. One grammar.”

#### State Singles

- **Contents**: Individual Line, Ray, and Bilateral glyphs in each native state color.
- **Role**: Low-cost impulse SKU and replacement/utility item.
- **Audience**: People who want to mark specific objects with specific functions.
- **Format**: Single die-cut stickers.
- **Copy**: “Choose the function. Choose the state. Mark the object.”

#### Mnemosyne Alphabet Sheet

- **Contents**: 22 glyphs in a single chart format.
- **Role**: Collector SKU and language-reference object.
- **Audience**: Fans of the system, game designers, lore collectors.
- **Format**: Sticker sheet, poster, or foil card sheet.
- **Copy**: “Twenty-two operators arranged across the arc.”

#### Cipher Card Insert

- **Contents**: 0-2-1 system, primitive meanings, game verbs, composition rules.
- **Role**: Education layer included with every order.
- **Audience**: Everyone.
- **Format**: Fold-out card, postcard, or mini-zine.
- **Copy**: “This is not a logo sheet. It is a grammar key.”

### Second-wave tier

#### Operator Cards

- **Contents**: One card per glyph with semantic and game-facing function.
- **Role**: Premium collector/game bridge.
- **Format**: Trading-card deck or oracle-like reference deck without drifting into generic mysticism.
- **Copy**: “A deck of symbolic actions.”

#### Patch Set

- **Contents**: Embroidered primitives and selected high-frequency glyphs.
- **Role**: Apparel/accessory bridge.
- **Format**: Sew-on or iron-on patches.
- **Copy**: “Mark your field kit.”

#### Field Kit Bundle

- **Contents**: Primitive Pack, State Singles, Cipher Card, one patch, one alphabet sheet.
- **Role**: Flagship launch bundle.
- **Format**: Envelope or small archival box.
- **Copy**: “Everything needed to begin marking state.”

## Colorway lines

### Native Velmic

Primary system line. Use for the first launch because it carries the 0-2-1 state logic.

- **Best for**: state markers, rules, game components, core identity.
- **Tone**: active, technical, canonical.
- **Copy**: “The native register. Blue holds. Amber interrupts. Cyan activates.”

### Howlbox

Saturated violet/green flagship line. Use as the expressive brand line.

- **Best for**: apparel, patches, premium stickers, faction identity.
- **Tone**: vivid, loudest allowable line.
- **Copy**: “The signal goes feral without losing its grammar.”

### Stellaria

Celestial black/star/copper line. Use as the elevated collector line.

- **Best for**: posters, foil cards, night-mode stickers, premium packaging.
- **Tone**: quiet, cosmic, precise.
- **Copy**: “A cold field. A copper seal. The glyph remains.”

### Neutral / Void / Umbra

Archival bone/ink/ochre line. Collapse overlapping neutral palettes into one practical product family.

- **Best for**: document seals, notebooks, zines, archival cards, apparel labels.
- **Tone**: minimal, canonical, artifact-like.
- **Copy**: “Filed, sealed, and ready to carry.”

## SKU naming system

Use names that encode product type, glyph family, state, finish, and colorway.

### Format

```text
VEL-[family]-[slot]-[state]-[finish]-[colorway]
```

### Examples

```text
VEL-PRIM-F02-1-CLEAR-NATIVE
VEL-PRIM-F03-2-SOLID-NATIVE
VEL-ALPH-G07-0-CLEAR-NATIVE
VEL-PACK-F01F04-021-MIXED-NATIVE
VEL-SHEET-G01G22-X-SOLID-STELLARIA
```

### State codes

- `0`: Blue / Carry / Hold
- `2`: Amber / Pause / Interrupt
- `1`: Cyan / Return / Activate
- `X`: Non-native decorative or collector colorway

### Finish codes

- `CLEAR`: translucent backing
- `SOLID`: opaque backing
- `MIXED`: combined finish pack
- `PATCH`: embroidered/sewn format
- `CARD`: card format
- `POSTER`: poster format

## Apparel direction

### Shirts

Keep shirts minimal. Use one large glyph on the back or one small glyph at the chest. Add metadata on sleeve or hem.

Good shirt concepts:

- **F02 Ray / Return State**: small cyan Ray on left chest, “COMMIT DIRECTION” on sleeve.
- **Bilateral Exchange**: back print with F03, small text “DIALOGUE / EXCHANGE / BRIDGE.”
- **0-2-1 Register Tee**: three glyphs stacked vertically with Blue, Amber, Cyan state labels.
- **Mnemosyne Chart Tee**: full 22-glyph chart on back, tiny “VELMIC / SYMBOL REFERENCE” on front.

### Hats

Use one primitive only. Embroidery should favor simple forms.

Best candidates:

- Line
- Ray
- Bilateral
- One confirmed high-frequency derived glyph

### Patches

Patches should feel like field markers or faction identifiers.

Patch shapes:

- Rectangle with slot metadata.
- Circular seal for state sets.
- Narrow strip patch for Line/Ray forms.
- Shield or tab only if it follows actual glyph geometry.

## Do and do not

### Do

- Treat every glyph as a functional unit.
- Include slot IDs wherever possible.
- Make the color-state relationship explicit.
- Use packaging to teach the grammar.
- Build game mechanics into the merch language.
- Keep layouts sparse and canonical.

### Do not

- Call the glyphs “runes” unless that is intentionally part of a separate lore layer.
- Overuse fake ancient texture or occult framing.
- Sell colorways as random aesthetic variants.
- Put too many glyphs on apparel without reference structure.
- Add unrelated icons, mascots, or flourishes.
- Let merch drift away from the language system.
