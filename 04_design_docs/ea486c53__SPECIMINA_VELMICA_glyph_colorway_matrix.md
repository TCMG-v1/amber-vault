## Praefatio

This matrix is the production scaffold for the Velmic-glyph die-cut sticker line — the first-priority MERCH track from the *Registrum Proprietatis*. It pairs each glyph (and each foundational form from which glyphs are composed) with the colorways available in the corpus: the **native Velmic palette** (0-2-1 system) plus four selected colorways from the *Codex Chromaticus*.

**What this scaffold supplies:** structure, colorway specs, SKU framing, production notes.
**What this scaffold does not yet supply:** the glyph art itself. Glyph designs live in the *Velmic Architecture Manual — Print Edition* (Drive: `Velmic Architecture Manual - Print Edition.PDF`, Appendix B: Symbol Reference Chart). Extracting them to production vector format (SVG preferred) is the gating task before a print run.

The Velmic system rests on four primitive forms from which all higher-order glyphs are composed. These are merch-viable on their own — clean line-art, ideal for die-cut, archetypally minimal — and could anchor a "Primitives" product line distinct from the full alphabet pack.

| F4 | **[fourth form]** | *(extract from PDF)* | *(closed / terminated form per pattern; confirm from Architecture Manual)* | TBD on form |

**Product framing — Primitives Pack:** A 4-glyph pack containing F1–F4 in all five colorways. Lower glyph count, simpler die-cut shapes, lower production cost, archetypal appeal. Possible entry-level SKU.

## § V — Production Notes

### V.a — Asset extraction (gating)

Glyph designs need to be extracted from `Velmic Architecture Manual - Print Edition.PDF` (Appendix B: Symbol Reference Chart) to production vector format. Three paths, in order of fidelity:

1. **Vector extraction from PDF.** If the PDF was generated from a vector source (likely — it has clean linework per the snippet), the glyph paths can be extracted directly with a PDF tool (Inkscape's PDF import, Adobe Illustrator's PDF open, or pdftk + further conversion). This preserves the original line weights and joins. **Try this first.**
2. **Manual re-tracing.** If extraction produces poor vectors, re-trace each glyph in Illustrator / Inkscape / Affinity Designer using the PDF as a reference. Slower, but produces clean production-ready SVG.
3. **Recomposition from rules.** If the four foundational forms and the compositional rules are documented in the Architecture Manual Chapter 1, the 22 glyphs could be programmatically composed via SVG generation. Highest fidelity to the *system*, but requires implementation work.

### V.b — Die-cut considerations

- Minimum line weight for die-cut survivability: roughly 0.5pt at 1.5" sticker size (scale up for larger sizes).
- Glyphs composed primarily of `──` Line and `──▸` Ray strokes should die-cut cleanly. Composed glyphs with internal closures (likely some of the derived 22) may need bridge points or solid-backing treatment to maintain integrity.
- The four foundational forms (F1–F4) are the easiest die-cut shapes; the alphabet glyphs (G01–G22) need per-glyph evaluation.

### V.c — Print test sequence

1. Home printer with vinyl sticker paper — prove the colorway selection and verify die-cut shape feasibility per glyph
2. Refine: adjust line weights, simplify any glyphs that fail die-cut
3. Single-color test order with a production vendor (Sticker Mule / StickerApp / local shop) — verify color accuracy at production scale
4. Production order for the launch SKU(s)

### V.d — Pricing posture

Estimate at production scale (assume Sticker Mule pricing tier, mid-2026):

- Single die-cut sticker, 3" max: ~$0.30–$0.50 unit cost at 500-unit order; retail $3–$5
- 22-glyph pack: ~$5–$8 unit cost at 250-pack order; retail $20–$35
- Premium pack (both finishes, all colors): ~$15–$25 unit cost at 100-pack order; retail $50–$80

Margins are healthy at every tier. Velmic-glyph stickers are entry-merch — they pay for themselves quickly and seed the higher-ticket items (Wellhouse print editions, font licensing, BTB decks).
