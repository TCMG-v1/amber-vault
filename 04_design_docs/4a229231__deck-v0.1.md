# Pixel Deck · v0.1 — Card Spec

10 distinct cards. 24 total card count with duplicates per game design.

Each card has:
- **Prime number** — its canonical index
- **Hanzi** — the CJK face
- **Pinyin + tone**
- **Gloss** — short English meaning
- **C face** — the function that IS the rule
- **Game rule** — the natural-language reading of the C
- **Etymon note** — short philological gloss (the Etymon's hand on each card)

---

## 0001 · 無 · wú · emptiness

```c
void *wu(void) {
    return NULL;
}
```

**Rule.** When you PLACE 無, take one card from the dormant pool and return it to the bottom of the draw pile. The void consumes from the table and returns nothing to it.

**Etymon note.** The character 無 derives from an oracle-bone pictograph of a dancer waving sleeves — emptiness as *performance*, the gesture that fills space by moving through it. The Daoist sense (the void as generative) came later; the original was kinetic. Compare Sanskrit *śūnya* (zero, void) and Arabic *ṣifr* (zero, also lent us "cipher"). All three civilizations independently arrived at the void as a positional placeholder. The C `NULL` is the most recent member of this lineage — emptiness as an *address that points nowhere*. *That which returns nothing has emptied the room.*

## 0003 · 值 · zhí · worth / value

```c
int zhi(int x) {
    return x;
}
```

**Rule.** When you PLACE 值, the next player's TRANSFORM action this round is free (does not consume their normal turn — they may TRANSFORM and then take another action). Worth is conserved across the transaction.

**Etymon note.** 值 combines 人 (person) with 直 (straight, upright) — value as *uprightness embodied in a person who can be trusted to weigh fairly*. The semantic field includes "to be worth," "to be on duty," "to be straightforward." The C `int zhi(int x) { return x; }` is the identity function — the purest expression of value-preservation. Compare Latin *valēre* (to be strong, to be worth — root of "value," "valid," "valor") and Sanskrit *artha* (purpose, wealth, meaning). All trace back to the question: *what does this exchange preserve?* *Worth is what survives the trade.*

## 0007 · 張 · zhāng · stretch / dilate / open

```c
double zhang(double t, double rho) {
    return t * rho;
}
```

**Rule.** When you PLACE 張, the dormant pool's *effective size* doubles for the next round — players may RECEIVE two cards from the pool instead of one (this is the only time RECEIVE may take two). Time dilates; abundance follows.

**Etymon note.** 張 (zhāng) is also the most common Chinese surname — but its core verb-sense is "to draw a bow," "to stretch open," "to extend." The radical 弓 (bow) is right there in the character. The English "tension" and Latin *tendere* (to stretch) share an Indo-European root *ten-* — every language has a word for the moment before release. In CHRONOS lore, this is ρ — the dilation coefficient. The C function multiplies time by rho. *The bow that is drawn is the bow that has not yet released.*

## 0011 · 岔 · chà · fork / divergence / branch

```c
int cha(int *a, int *b, int v) {
    *a = v; *b = v;
    return 2;
}
```

**Rule.** When you PLACE 岔, duplicate any one card currently in the dormant pool (take a second copy of it from the draw pile if available, or take one from another player's 化 pile if they have it — their choice which they surrender). The play forks the pool.

**Etymon note.** 岔 combines 山 (mountain) with 分 (divide) — a *fork in the mountain path*. The character is geographically literal: the place where one road becomes two. In English, "fork" itself comes from Latin *furca* (a pitchfork, a gallows, a forked branch). Every language that has roads has a word for the moment the road splits. In version control and CHRONOS's Looping Shatter Mode, the fork is the branch — and every fork creates a hash. The C function writes the same value to two pointers — the *forked write* — and returns 2, because that is what forking produces: two of what was one. *Every road that splits is the same road, twice.*

## 0013 · 碎 · suì · shatter / fragment

```c
void sui(void *p) {
    free(p);
}
```

**Rule.** When you PLACE 碎, any one player (your choice — including yourself) must discard one card from their hand to the bottom of the draw pile. The card is *freed* — it returns to the pool of potential.

**Etymon note.** 碎 has the 石 (stone) radical and the phonetic 卒 (soldier, end) — a *stone broken at its end*, a final shattering. The semantic field includes "to break into pieces," "fragmentary," "trivial" (as in: too small to gather). Compare Old English *scēadan* (to scatter, divide — root of "shed" and "shatter") and Sanskrit *śi-* (to break, cut). The C function `free(p)` is the canonical destructive act in C — it releases memory back to the heap, irrevocably. *What is shattered cannot be unshattered, but it can be cast again.*

## 0019 · 封 · fēng · seal / close / enfeoff

```c
const int feng(int x) {
    return x;
}
```

**Rule.** When you PLACE 封, declare one specific card type in the dormant pool *sealed against RECEIVE* until your next turn — no player may RECEIVE that card type for one round. The seal holds.

**Etymon note.** 封 combines 圭 (jade tablet) with 寸 (a hand's-breadth, an inch) — historically, the feudal *enfeoffment* — the lord's hand granting jade tablets to mark sealed territory. The character carries the full weight of medieval Chinese statecraft. In modern usage: to seal a letter, to seal a vow, to seal a tomb. Compare Latin *sigillum* (a small image, a seal — root of "sigil") and Arabic *khatm* (seal, also "finality" — the Khatm-al-anbiyā, the seal of the prophets). The C function with `const` return preserves the value against mutation — a typed seal. *That which is sealed has accepted its shape.*

## 0023 · 道 · dào · way / path / method

```c
void (*dao)(void) = NULL;
```

**Rule.** When you PLACE 道, all players (including you) reveal one card from their 專 latched hand to the table, face-up, for the duration of one round. After the round, hands return to face-down. The way is briefly visible.

**Etymon note.** 道 needs no introduction in this household. The character combines 辶 (motion, walking) with 首 (head) — *the head moves along the path*. The Daoist sense (the Way as cosmic principle) is layered over the literal sense (a road, a method, a doctrine). Compare Sanskrit *mārga* (path, way — also "marg" in modern Hindi), Greek *hodos* (road — root of "method," literally "the way after"), and the Arabic *ṭarīqa* (path, especially a Sufi order's path). The C declaration is a function pointer pointing to NULL — *the way that can be named is not the eternal way; the way that is null may still be walked.* *The way is not the destination, and yet you arrive.*

## 0029 · 養 · yǎng · nurture / hold dormant / sustain

```c
void yang(int *seed) {
    while (*seed > 0) (*seed)--;
}
```

**Rule.** When you PLACE 養, you may move one card from any player's 化 sealed pile back to the dormant pool (養 state). The seal is decomposed back into nurture. (This is a powerful play — use sparingly. The chord-tracker rule: this play breaks chords.)

**Etymon note.** 養 has the 食 (food) radical at the bottom and 羊 (sheep) above — the *feeding of sheep*, agricultural sustenance. The semantic field grew from "to feed" to "to nurture" to "to cultivate" to "to be raised by." Modern Mandarin uses 養成 (yǎngchéng) for "to develop a habit" — habits are nurtured the way livestock are. Compare Latin *alere* (to nourish — root of "alimony," "aliment") and Sanskrit *poṣ-* (to thrive, nourish — root of *pushti*). The C function decrements a seed until empty — the slow consumption that is also growth, the sheep that eat the field but in eating cultivate it. *To nurture is to consume slowly enough that the consuming becomes growth.*

## 0031 · 化 · huà · transform / become / change

```c
void *hua(void *src, size_t n) {
    void *dst = malloc(n);
    memcpy(dst, src, n);
    return dst;
}
```

**Rule.** When you PLACE 化, choose any two cards in the dormant pool and swap their effective hanzi for the next round — the cards keep their physical face but their *rules* swap. (The Etymon notes: in transformation, the form changes but the substance persists, or sometimes vice versa.)

**Etymon note.** 化 is one half of the binary 變化 (biànhuà, change), and one half of the central Daoist concept 造化 (zàohuà, creation-transformation, the working of nature). The character originally depicted a person upright (人) and a person inverted — *one figure becoming another by rotation*. Transformation as topology, not destruction. Compare Greek *metamorphōsis* (literally "form-after-form") and Sanskrit *pariṇāma* (transformation, evolution — central to Sāṃkhya philosophy). The C function allocates new memory and copies the source into it — the canonical transformation: same content, new substrate. *To transform is to be copied into a new substrate without ceasing to be yourself.*

## 0037 · 專 · zhuān · latch / focus / concentrate

```c
volatile int zhuan = 1;
while (zhuan) { /* hold */ }
```

**Rule.** When you PLACE 專, you skip your *next* turn but draw 3 cards immediately. The latch holds — you commit deeply, paying with time, gaining with hand-strength. (House rule note: in beginner play, "skip next turn" feels harsh — beta version may soften to "next turn, you may only HOLD.")

**Etymon note.** 專 combines 寸 (hand's-breadth) with 叀 (spindle, a thing wound tightly) — the *hand that winds the spindle*, focus as the act of tight winding. Modern usage: 專心 (zhuānxīn, concentrated heart), 專家 (zhuānjiā, specialist). The semantic field is wholly about *narrowing toward depth*. Compare Latin *focus* (literally "hearth" — root of English "focus" — the gathered point where the fire concentrates) and Sanskrit *dhāraṇā* (concentration, one of the limbs of yoga). The C snippet is a busy-wait loop — the CPU holds, polling, refusing to release until the volatile flag flips. The deepest latch is the one that won't yield. *To latch is to refuse all other doors until the chosen door opens.*
